## DROP

Podemos usar DROP para eliminar tablas y bases de datos del servidor

```
mysql> DROP TABLE logins;

Query OK, 0 rows affected (0.01 sec)


mysql> SHOW TABLES;

Empty set (0.00 sec)
```

## ALTER

Podermos usar ALTER para cambiar el nombre de cualquier tabla y cualquiera de sus campos o para eliminar o agregar una nueva columna a una tabla existente.
El siguiente ejemplo agrega una columna. **newColumn** hacia **logins** table usando ADD.

```
mysql> ALTER TABLE logins ADD newColumn INT;

Query OK, 0 rows affected (0.01 sec)
```

Para cambiar el nombre de una columna, podemos usar **RENAME COLUMN**:
```
mysql> ALTER TABLE logins RENAME COLUMN newColumn TO oldColumn;

Query OK, 0 rows affected (0.01 sec)
```

Tambien podemos cambiar el tipo de datos de una columna con **MODIFY**
```
mysql> ALTER TABLE logins MODIFY oldColumn DATE;

Query OK, 0 rows affected (0.01 sec)
```

Finalmente podemos eliminar una columna usando DROP
```
mysql> ALTER TABLE logins DROP oldColumn;

Query OK, 0 rows affected (0.01 sec)
```

## Clasificacion de resultados

Podemos ordenar los resultados de cualqueir consulta usando **ORDER BY**

```
mysql> SELECT * FROM logins ORDER BY password;

+----+---------------+------------+---------------------+
| id | username      | password   | date_of_joining     |
+----+---------------+------------+---------------------+
|  2 | administrator | adm1n_p@ss | 2020-07-02 11:30:50 |
|  3 | john          | john123!   | 2020-07-02 11:47:16 |

4 rows in set (0.00 sec)
```

Por defecto la clasificacion se realiza en orden ascendente, pero tambien podemos ordenar los resultados por **ASC** o **DESC**

```

```
mysql> SELECT * FROM logins ORDER BY password DESC, id ASC;

## LIMIT

En caso de que nuestra consulta devuelva una gran cantidad de registros, podemos limitar los resultados a los que queremos unicamente, utilizando LIMIT y el numero de registros que queremos:

```
mysql> SELECT * FROM logins LIMIT 2;

+----+---------------+------------+---------------------+
| id | username      | password   | date_of_joining     |
+----+---------------+------------+---------------------+
|  1 | admin         | p@ssw0rd   | 2020-07-02 00:00:00 |
|  2 | administrator | adm1n_p@ss | 2020-07-02 11:30:50 |
+----+---------------+------------+---------------------+
2 rows in set (0.00 sec)
```

Si queremos limitar los resultados con un desplazamiento, podriamos especificar el desplazamiento antes del recuento LIMITE:

```
mysql> SELECT * FROM logins LIMIT 1, 2;

+----+---------------+------------+---------------------+
| id | username      | password   | date_of_joining     |
+----+---------------+------------+---------------------+
|  2 | administrator | adm1n_p@ss | 2020-07-02 11:30:50 |
|  3 | john          | john123!   | 2020-07-02 11:47:16 |
+----+---------------+------------+---------------------+
2 rows in set (0.00 sec)

```

## LIKE

Otra clausula util es LIKE, que permite seleccionar registros haciendo coincidir un patron determinado.

```
mysql> SELECT * FROM logins WHERE username LIKE 'admin%';

+----+---------------+------------+---------------------+
| id | username      | password   | date_of_joining     |
+----+---------------+------------+---------------------+
|  1 | admin         | p@ssw0rd   | 2020-07-02 00:00:00 |
|  4 | administrator | adm1n_p@ss | 2020-07-02 15:19:02 |
+----+---------------+------------+---------------------+
2 rows in set (0.00 sec)
```
El % actua como comodin y coincide con todos los caracteres despues de admin

```
mysql> SELECT * FROM logins WHERE username like '___';

+----+----------+----------+---------------------+
| id | username | password | date_of_joining     |
+----+----------+----------+---------------------+
|  3 | tom      | tom123!  | 2020-07-02 15:18:56 |
+----+----------+----------+---------------------+
1 row in set (0.01 sec)
```

El simbolo _ se utiliza para hacer coinicir exactamente un caracter, como el ejemplo donde coincide con tres caracteres al igual que tom
